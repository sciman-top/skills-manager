# Security Policy

[English](#english) | [中文](#中文)

## 中文

### 支持范围

默认仅对 `main` 分支最新提交提供安全修复支持。

### 漏洞报告

如发现安全问题，请不要公开提交 issue。

请私下联系维护者，并尽量提供：

1. 漏洞描述与影响范围
2. 最小复现步骤
3. 受影响版本或提交范围
4. 可选的修复建议

维护者会在确认后评估影响、安排修复，并在适当时机披露。

### 披露原则

在修复发布前，请避免公开披露可被直接利用的细节。

## English

### Supported Scope

Security fixes are provided for the latest commit on the `main` branch by default.

### Reporting a Vulnerability

Please do not open a public issue for security problems.

Contact the maintainer privately and include, when possible:

1. a description of the issue and impact
2. minimal reproduction steps
3. affected versions or commit range
4. an optional remediation suggestion

The maintainer will validate the report, assess impact, and coordinate a fix before public disclosure.

### Disclosure

Please avoid publicly disclosing exploitable details before a fix is available.
