# Cross-Platform Artifact Policy

> Standard for handling OS-specific artifacts and platform tooling across workspace repos.
>
> Version: 1.0.0 | Date: 2026-03-31 | Issue: #1535

---

## OS Residue (ALWAYS IGNORE)

These files are auto-generated by operating systems and must never be committed.

| Pattern | OS | Purpose |
|---------|----|---------|
| `.DS_Store` | macOS | Finder metadata |
| `Thumbs.db` | Windows | Thumbnail cache |
| `desktop.ini` | Windows | Folder display settings |
| `._*` | macOS | Resource forks |
| `$RECYCLE.BIN/` | Windows | Recycle bin |
| `.Spotlight-V100` | macOS | Spotlight index |
| `.Trashes` | macOS | Trash metadata |

**Coverage:** All patterns are in the workspace root `.gitignore`. Each child repo should also ignore these in its own `.gitignore`.

## Platform-Specific Scripts (INTENTIONAL — DO NOT IGNORE)

These file types contain intentional cross-platform tooling:

| Extension | Status | Notes |
|-----------|--------|-------|
| `.bat` | **Keep** | Windows batch scripts for setup, builds, automation |
| `.ps1` | **Keep** | PowerShell scripts for cross-repo operations |
| `.cmd` | **Keep** | Windows command scripts (none currently present) |
| `.sh` | **Keep** | Unix shell scripts |

### Rules for Platform Scripts

1. Platform scripts MUST contain real functionality (not empty stubs)
2. Scripts named `temp*.bat` or similar should be cleaned up after use
3. If a script exists for one platform, a cross-platform equivalent is recommended but not required
4. Document platform-specific scripts in the repo's README if they are part of the user workflow

## Gitignore Baseline

Every repo in the workspace ecosystem should include at minimum:

```gitignore
# OS artifacts
.DS_Store
Thumbs.db
desktop.ini
._*
```

## Exception Handling

If a repo intentionally needs to commit an OS artifact (unlikely but possible):
1. Add a comment in `.gitignore` explaining the exception
2. Use `git add -f <file>` to force-add
3. Document the reason in the repo's AGENTS.md or docs/
