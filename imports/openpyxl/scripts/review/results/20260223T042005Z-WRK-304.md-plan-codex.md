### 4. `consume-signals.sh` — signal contract

The simplified script must meet this contract:

- **Input:** JSON from stdin with at minimum `session_id` (string), `timestamp` (ISO8601)
- **Output:** JSONL entry appended atomically to `.claude/state/session-signals/YYYY-MM-DD.jsonl`
  using `>>` (append is atomic for lines < PIPE_BUF on Linux/macOS)
- **Invalid input:** if `session_id` is absent or stdin is empty, write a minimal stub entry
  with `session_id: "unknown-<epoch>"` and continue; never exit non-zero on input errors
- **Naming collision:** YYYY-MM-DD bucketing means multiple sessions per day append to the
  same file — this is intentional; Phase 1 reads all entries by date range
