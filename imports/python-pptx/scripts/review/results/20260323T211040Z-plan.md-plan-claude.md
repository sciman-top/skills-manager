- [P3] Minor: Test plan has no test for `su_average(z1, z2)` directly — consider adding a dedicated unit test for the depth-averaged Su calculation, especially verifying z1 > z2 ordering.
- [P3] Minor: The YAML calculation report (AC #6) has no schema sketch or example keys listed — a brief outline (e.g., `inputs:`, `methodology:`, `results:`, `fos_check:`) would help reviewers validate the output format during implementation.

### Suggestions
- Add a brief note on units consistency (kPa, kN, m) as a docstring or comment convention — geotechnical code is notorious for unit-mismatch bugs.
- Consider adding a `__post_init__` validation on `SoilProfile` to reject negative `su_0_kpa` or `k_kpa_per_m` values, complementing the negative-depth test.
- The Nc interpolation for shallow embedment (`depth <= 4.5 * width`) is mentioned but not detailed — a one-liner referencing the specific DNV-RP-E302 figure/table would help the implementer.
- Test #2 (shallow Nc interpolation) could specify the expected interpolation method (linear between Nc=0 at surface and Nc_deep at 4.5B) to avoid ambiguity.
