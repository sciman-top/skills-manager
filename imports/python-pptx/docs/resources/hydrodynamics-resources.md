## GitHub Repositories

- [The Well (github)](https://github.com/PolymathicAI/the_well) — 15 TB physics simulation dataset covering fluid dynamics, acoustics, MHD, reaction-diffusion, and astrophysics. CC BY 4.0 data license permits commercial use with attribution. HF streaming via WellDataset avoids bulk download. Pretrained FNO/TFNO/UNet baselines available per dataset on Hugging Face (updated March 2025). Smoke test passed on dev-secondary (uv, Python 3.11).

- [wavespectra (Ocean Wave Spectra Library)](https://github.com/wavespectra/wavespectra) — xarray-based Python library for ocean wave spectral data. v4.4.2 (March 2025). 15+ format readers: WW3, SWAN, ERA5, NDBC. 60+ spectral parameter methods. Partitioning: PTM1-5, watershed, wave age. Directly fills wave_spectra gap in digitalmodel/hydrodynamics and metocean module (see WRK-383). MIT, 20 contributors.

- [BEMRosetta](https://github.com/BEMRosetta/BEMRosetta) — C++ with GUI. More actively maintained than Bemio for BEM data handling. Can export to formats readable by Python. Useful as a standalone validation tool alongside the ACE Python pipeline.

- [Bemio](https://github.com/WEC-Sim/bemio) — Python-based. Note: requires Python 2.7 (legacy project). Consider using Capytaine's built-in I/O or BEMRosetta as more actively maintained alternatives for BEM data handling.

- [Capytaine](https://github.com/capytaine/capytaine) — pip-installable, native Python API. Outputs directly to xarray datasets. Results feed into OpenFAST/RAFT/WEC-Sim for time/frequency-domain analysis. NREL-funded development ensures alignment with OpenFAST ecosystem.

- [HAMS](https://github.com/YingyiLiu/HAMS) — Fortran-based, driven via input files. Python wrapper (pyHAMS) available for integration. Can be used as alternative to Capytaine; compare results for validation. Apache license more permissive than Capytaine's GPL.

- [OceanWave3D](https://github.com/apengsigkarup/OceanWave3D-Fortran90) — Fortran 90 code; no Python API. Drive via subprocess and file I/O. Can couple with OpenFOAM via waves2Foam for combined wave generation and CFD. DTU-developed research tool.

- [WEC-Sim](https://github.com/WEC-Sim/WEC-Sim) — MATLAB/Simulink-based; no direct Python API. Uses BEM data from Capytaine/HAMS/WAMIT. For Python-native workflows, consider using Capytaine + custom time-domain solvers or OpenFAST HydroDyn module instead.
