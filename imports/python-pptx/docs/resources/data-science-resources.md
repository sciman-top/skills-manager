## GitHub Repositories

- [Dask](https://github.com/dask/dask) — Drop-in parallel replacement for pandas/NumPy. Dask DataFrames partition large worldenergydata CSV/Parquet files. Dask arrays enable large-scale matrix operations in digitalmodel FEA.

- [NumPy](https://github.com/numpy/numpy) — Already a core dependency across the ACE ecosystem. Standard array interchange format. Basis for SciPy, Pandas, and all scientific Python libraries used in ACE.

- [Pandas](https://github.com/pandas-dev/pandas) — Already a core dependency. Standard data interchange format in the ACE ecosystem. Parquet serialization for efficient storage. Excel I/O for engineering spreadsheet integration.

- [Polars](https://github.com/pola-rs/polars) — Drop-in replacement for pandas in data-heavy pipelines. Arrow-based interchange with pandas via .to_pandas() / .from_pandas(). Consider for worldenergydata ETL pipelines processing large BSEE/SODIR datasets.

- [PyArrow](https://github.com/apache/arrow) — Backend for pandas Parquet I/O and Polars data interchange. Dataset API for reading partitioned worldenergydata archives. Arrow Flight for potential data service architecture.

- [Rasterio](https://github.com/rasterio/rasterio) — Pythonic wrapper over GDAL raster functionality. NumPy array interface for raster data. Integrates with xarray via rioxarray extension for multi-dimensional analysis.

- [Shapely](https://github.com/shapely/shapely) — Based on GEOS library. Direct NumPy integration via vectorized operations. Foundation geometry type for all GIS operations in the ACE ecosystem.

- [Vaex](https://github.com/vaexio/vaex) — HDF5 and Arrow backends for zero-copy data access. Useful for interactive exploration of large BSEE datasets. Note: community is smaller than Polars/Dask; evaluate long-term maintenance.

- [Xarray](https://github.com/pydata/xarray) — Standard format for climate/ocean data (NetCDF via xarray). Dask backend enables out-of-core processing. rioxarray extension adds geospatial capabilities. Ideal for metocean data in worldenergydata.

- [lasio (LAS File I/O for Well Logs)](https://github.com/kinverarity1/lasio) — Python 3.7+ library for reading/writing LAS 1.2 and 2.0 files (borehole logs). Pandas DataFrame integration. Active releases (v0.32). Companion library welly adds curve/well/project management. Critical for well log data ingestion in worldenergydata.
