    Replace the stub in `bsee_refresh.py` with a wired implementation:

    1. Import `BSEEWebScraper` from `worldenergydata.bsee.data.scrapers.bsee_web`, `zipfile`, `io`, `pandas as pd`, `Path` from pathlib.

    2. Define a module-level dict of BSEE dataset types to process per D-10:
    ```python
    BSEE_DATASETS = {
        "platform": {"url_key": "platform", "output_file": "bsee_platform_structures.parquet"},
        "pipeline_permit": {"url_key": "pipeline_permit", "output_file": "bsee_pipeline_permits.parquet"},
        "pipeline_location": {"url_key": "pipeline_location", "output_file": "bsee_pipeline_locations.parquet"},
        "deepwater_structure": {"url_key": "deepwater_structure", "output_file": "bsee_deepwater_structures.parquet"},
    }
    ```

    3. In `BseeRefreshJob.run(self, config)`:
       a. Get `output_dir = Path(config.get("output_dir", "data/bsee"))` and create it
       b. Create `scraper = BSEEWebScraper()`
       c. Loop over `BSEE_DATASETS`. For each dataset:
          - Get URL from `BSEEWebScraper.URLS[info["url_key"]]`
          - Call `scraper.download_zip_to_memory(url, data_type=info["url_key"])`
          - If None returned, log warning and continue to next dataset
          - Extract first CSV from zip using `zipfile.ZipFile(io.BytesIO(zip_bytes))`
          - Read CSV into DataFrame: `pd.read_csv(zf.open(csv_name))`
          - Write Parquet: `df.to_parquet(output_dir / info["output_file"], engine="pyarrow", index=False, compression="snappy")`
          - Track rows: add `len(df)` to running total
       d. If total_records > 0: return success JobResult
       e. If no datasets succeeded: return failure JobResult with error listing which downloads failed

    4. Anti-pattern avoidance per RESEARCH.md:
       - Do NOT put all downloads in a single try/except — each dataset independently handled
       - Do NOT add retry logic inside the adapter — RetryManager handles that at scheduler level
       - Do NOT hardcode paths — use `config.get("output_dir")` or Settings.data_dir
       - Use stdlib `logging` (not loguru) to match scheduler patterns

    5. Create `test_bsee_adapter.py`:
       - Mock `BSEEWebScraper.download_zip_to_memory` to return fake zip bytes containing a small CSV
       - Use a helper that creates an in-memory zip with a CSV file: `io.BytesIO` + `zipfile.ZipFile`
       - Test all 6 behaviors using `tmp_path` fixture for output directory
       - For failure tests, have mock return `None` for specific dataset types
