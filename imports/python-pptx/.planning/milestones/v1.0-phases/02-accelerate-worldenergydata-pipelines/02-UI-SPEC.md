# Phase 02 — UI Design Contract

> Visual and interaction contract for the pipeline monitoring dashboard. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope note:** Phase 02 is primarily a backend/data pipeline phase. The UI component is a single status page added to the existing `monitoring-dashboard` frontend (`monitoring-dashboard/frontend/`). This contract extends the existing design system rather than creating a new one.

## Component Inventory

This phase adds one new page and reuses existing components.

### New Page: Pipeline Status (`/pipelines`)

**Layout:** Sidebar + main content (reuse existing `Layout.tsx` shell).

**Primary focal point:** Source Status Grid (Section 1). This is the first thing users see and the primary orientation element on the page. It occupies the top of the content area, uses the widest layout span, and each card carries a colored left border accent to draw visual attention to source health at a glance.

#### Section 1: Source Status Grid
- Grid of `MetricCard` components (reuse existing), one per data source
- Each card shows: source name, last successful run (monospace timestamp), records count, status badge
- Grid layout: `grid-cols-metrics` (existing `repeat(auto-fit, minmax(250px, 1fr))`)
- 7 cards total (one per configured source: EIA, BSEE, SODIR, Metocean, Brazil ANP, UKCS, LNG Terminals)

#### Section 2: Freshness Timeline
- Horizontal bar chart showing time since last successful run per source
- Threshold markers showing staleness boundaries (SODIR: 36h, BSEE: 10d, EIA: 45d)
- Uses existing `recharts` BarChart or custom D3 (follow existing `LineChart.tsx` pattern)
- Colors: success (within threshold), warning (75-100% of threshold), error (exceeded threshold)

#### Section 3: Recent Job Runs
- Table with columns: Source | Status | Started | Duration | Records | Error
- Timestamps in monospace font (JetBrains Mono)
- Status column uses existing `status-badge` component classes
- Error column truncated with tooltip on hover (reuse existing `.tooltip` class)
- Sorted by most recent first, limited to last 20 runs

#### Section 4: Alert History
- Reuse existing `AlertsWidget.tsx` pattern
- Shows recent email alerts with severity badges (`.alert-critical`, `.alert-medium`, `.alert-low`)
- Each alert shows: timestamp, source name, alert type (failure / staleness), message

## Data Contract (status.json mapping)

The UI reads from `status.json` produced by the existing `StatusReporter`. Expected shape:

```typescript
interface PipelineStatus {
  jobs: Record<string, {
    last_run: string;         // ISO 8601 timestamp
    status: 'success' | 'failure' | 'running';
    records_updated: number;
    error_msg: string | null;
    duration_seconds: number;
  }>;
  staleness: Record<string, {
    threshold_hours: number;
    is_stale: boolean;
    hours_since_last_success: number;
  }>;
  alerts: Array<{
    timestamp: string;
    job_name: string;
    type: 'failure' | 'staleness';
    message: string;
  }>;
}
```

## Interaction States

### Source Card States

| State | Visual Treatment |
|-------|-----------------|
| Fresh (within threshold) | Success badge, green left border accent |
| Aging (75-100% of threshold) | Warning badge, amber left border accent |
| Stale (exceeded threshold) | Error badge, red left border accent, `animate-pulse-slow` on badge |
| Running | Info badge with `animate-pulse`, spinner icon |
| Never run | Muted badge, dashed border |

### Manual Refresh Button

| State | Visual Treatment |
|-------|-----------------|
| Idle | Primary color outline button, "Refresh now" |
| Loading | Disabled, spinner replaces icon, "Refreshing..." |
| Success | Briefly show success toast via `react-hot-toast` (existing dependency) |
| Error | Error toast with message |

### Loading States

| Scenario | Treatment |
|----------|-----------|
| Initial page load | Skeleton cards (8px border-radius, muted background, pulse animation) |
| Data refresh | Cards remain visible, subtle overlay with `.loading-spinner` (existing class) |
| Connection lost | `ConnectionStatus` component shows "Disconnected" in error color |
