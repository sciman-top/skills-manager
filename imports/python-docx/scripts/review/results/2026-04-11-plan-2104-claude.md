## Issues Found

### MINOR-1: Weekly validation check specificity

The plan should clarify the relationship between #2096's weekly checklist (Section 7) and the new entry-point checks proposed here. Suggest adding a note in the "Files to Change" section for the weekly review template that specifies: "Incorporate #2096 Section 7 D.4 and D.6 items, plus add link-resolution checks for the new intelligence section in docs/README.md and docs/document-intelligence/README.md."

### MINOR-2: PR structure recommendation

Given 10+ files changing, the implementation should group commits logically:
1. Create `docs/document-intelligence/README.md` + update `docs/README.md` (core navigation)
2. Update wiki `CLAUDE.md` files (5 files, same change)
3. Update capabilities summary + weekly review template
4. Move/clean misplaced files + add superseded note

This is an implementation detail but noting it prevents a messy PR.

### MINOR-3: Approximate counts should cite source

The plan uses "19K+ pages" and "1M+ indexed documents." These come from `#2096` Section 4. The implementation should cite the accessibility map or the underlying indexes as the source for these counts in the navigation pages, per Navigation Standard 4 (no provenance invention).
