- [P1] Preserve wind spectrum settings in template — D:/workspace-hub/digitalmodel/docs/modules/orcaflex/library/templates/calm_buoy_moored/spec.yml:56-61
  The new `wind` section specifies `type: "NPD spectrum"` (plus `components` and `seed`), but these keys are not consumed by the modular schema/builders for this spec format, so generation falls back to constant wind instead of spectral wind. This materially changes the applied environmental loading for anyone using this template expecting stochastic NPD wind behavior.

- [P2] Encode current rotation values instead of comments — D:/workspace-hub/digitalmodel/docs/modules/orcaflex/library/templates/calm_buoy_moored/spec.yml:53-54
  The template comments state a rotated current profile (`[0,20,40]` deg), but the actual `profile` data only contains `[depth, factor]`, so generated models use zero rotation at all levels. That silently drops directional veering effects from the source C06 setup, reducing fidelity of current loading whenever this template is used as-is.
