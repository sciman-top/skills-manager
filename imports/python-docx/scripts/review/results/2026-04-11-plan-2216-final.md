## Summary of What the Plan Establishes

### 1. Collection characterization
`/mnt/ace/acma-codes` is a **mixed staging area** containing:
- **Net-new content:** OCIMF tandem mooring guidelines, CSA Z276 LNG standards (entirely new to ecosystem)
- **Overlapping content:** API RP 1111 (3rd Ed) and API RP 2SK (2nd Ed) — likely duplicate copies of documents already indexed from `/mnt/ace/0000 O&G/`
- **Non-document artifacts:** Thumbs.db and possibly other non-standard files
- **Partially-new content:** OCIMF MEG 2008 (3rd Ed predecessor of MEG4 already in wiki)

### 2. Recommended integration model
Register as a **new mounted source** (`acma_codes_local`) in the existing registry infrastructure, then:
- Dedup API content against existing `og_standards_local` corpus via `doc_key` matching
- Fully index and summarize genuinely new content (OCIMF tandem mooring, CSA Z276)
- Promote LLM-wiki pages for 3 new standards (tandem mooring, CSA Z276.1, CSA Z276.18)
- Update OCIMF MEG4 wiki page with historical edition context

### 3. Dedup/provenance risks
| Risk | Severity | Mitigation |
|---|---|---|
| API files may be identical or different scans | Low | doc_key hashing resolves automatically |
| OCIMF 2008 vs 2018 are distinct editions | Low | Link via superseded_by, not merge |
| Unknown additional content beyond sample | Medium | Phase A will discover; plan may need scope expansion |
| Sandbox blocked live enumeration | Medium | Implementation must verify full inventory |

### 4. Follow-on issue split recommendation
**Split into 4 issues** with clean dependency chain:
1. Source registration + Phase A indexing + dedup
2. Transfer ledger + provenance backfill
3. LLM-wiki promotion (3 new pages + 1 update)
4. Accessibility and entry-point updates
