## MINOR Findings (non-blocking)

1. **Add default classification rule.** When issue labels are missing or ambiguous, the contract should explicitly state: "default to General class." This prevents agents from skipping retrieval because they can't classify the issue.

2. **Add promotion prompt to closeout.** The closeout stage asks agents to identify promotion candidates per #2209 criteria, but doesn't provide a concrete prompt. Consider adding: "Ask yourself: did this issue produce any finding that would help future issues or wiki readers?" to make the judgment call easier.

3. **Clarify multi-class union behavior.** Risk #5 mentions multi-class issues should use the union of bundles, but this should be in the main contract body (not just Risks), since it affects routine behavior.

4. **Consider adding a "retrieval contract version" field.** As the contract evolves (e.g., when #2104/#2136 land), plans should note which version of the retrieval contract they followed. This enables audit of whether pre-contract plans should be held to contract standards. Low priority — can be added during implementation.
