1. The Problem with Standard AI / RAG Workflows
   - Most AI sessions are stateless: you ask, you get an answer, it's gone.
   - Retrieval-Augmented Generation (RAG) works but has no memory or compounding.
   - Every session starts from zero; no knowledge accumulates.

2. Andre Karpathy's LLM Wiki Concept (published April 4, 2026)
   - 5,000+ GitHub stars in under 48 hours.
   - Instead of raw document retrieval, the AI builds and maintains a persistent
     structured wiki -- a folder of interlinked Markdown files.
   - AI reads new sources, extracts key info, integrates it into existing pages,
     updates summaries, and flags contradictions.
   - Metaphor: LLM is the programmer; the wiki is the codebase.

3. Hermes Agent by Nous Research
   - Open-source AI agent with 26,000+ GitHub stars.
   - Tagline: "The agent that grows with you."
   - Built-in learning loop: creates skills from experience, improves over sessions,
     builds a model of the user over time.
   - Runs on a $5 VPS or GPU cluster.
   - Supports 200+ models via OpenRouter; integrates with Telegram, Discord,
     Slack, WhatsApp, Signal.
   - Has a skill system -- a library of reusable, improving procedures.

4. The Integration: Hermes Agent + LLM Wiki
   - Latest Hermes update ships Karpathy's LLM Wiki as a built-in skill.
   - Command: `hermes update` then `/om wiki <your research topic>`.
   - Demonstrated by Nous Research to study their own projects, papers, and code.

5. How LLM Wiki Works -- The Three Layers
   - Layer 1: Raw Sources -- documents, articles, papers. Immutable. AI reads only.
   - Layer 2: The Wiki -- AI-generated Markdown files: summaries, entity pages,
     concept pages, comparisons. AI writes, humans read.
   - Layer 3: The Schema -- configuration doc that defines wiki structure and rules.
     Evolved together by user and agent.

6. Three Core Operations
   - INGEST: Drop a new source; AI extracts info, writes/updates pages (1 source
     can touch 10-15 pages -- the compounding effect).
   - QUERY: Ask questions; AI searches wiki and returns synthesized answers with
     citations. Good answers can be saved back as new wiki pages.
   - LINT: AI health-checks the wiki for contradictions, stale info, orphan pages,
     and suggests new sources or questions.

7. Use Cases
   - Researchers: compound knowledge across papers over weeks/months.
   - Content Creators: research vault that grows instead of resetting per topic.
   - Developers: codebase + docs + papers all organized in one place.
   - Business Teams: feed Slack threads, meeting notes, and call transcripts into
     a living wiki, no manual maintenance required.

8. Practical Setup Tips
   - Install Hermes on Linux, macOS, or WSL2 with one command.
   - Run `hermes update` to get the LLM wiki skill.
   - Use Obsidian (free) as the viewer; graph view shows page connections.
   - Use the Obsidian Web Clipper extension to capture web articles as Markdown.
   - Two key navigation files: INDEX (catalog with one-line summaries) and LOG
     (chronological record of agent actions).
   - Invest time in your schema -- it is the rulebook and determines wiki quality.

9. Historical Connection: Vannevar Bush's Memex (1945)

     with associative trails.
   - The missing piece Bush never solved: who does the maintenance.
     Answer now: an AI agent.
