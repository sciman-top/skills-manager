    queue/

    **Step 2 — Create `queue/job-schema.yaml`:**
    Document the job format with examples:
    ```yaml
    # Solver Job Schema
    # Submit jobs by creating a YAML file in queue/pending/
    # Filename convention: {timestamp}-{name}.yaml

    schema:
      required:
        solver: "orcawave | orcaflex"
        input_file: "relative path to .owd or .dat input file"
      optional:
        output_dir: "where to save results (default: queue/completed/{job-name}/)"
        export_excel: true  # also produce .xlsx alongside .owr
        description: "human-readable job description"

    example:
      solver: orcawave
      input_file: docs/domains/orcawave/L00_validation_wamit/2.1/OrcaWave v11.0 files/test01.owd
      export_excel: true
      description: "L00 validation smoke test"
    ```

    **Step 3 — Create `scripts/solver/submit-job.sh`:**
    ```bash
    #!/usr/bin/env bash
    # ABOUTME: Submit a solver job to the queue — creates YAML, commits, pushes
    set -euo pipefail

    REPO_ROOT="$(git rev-parse --show-toplevel)"
    SOLVER="${1:?Usage: submit-job.sh <solver> <input_file> [description]}"
    INPUT_FILE="${2:?Usage: submit-job.sh <solver> <input_file> [description]}"
    DESCRIPTION="${3:-Solver job}"
    TIMESTAMP="$(date -u +%Y%m%dT%H%M%SZ)"
    JOB_NAME="${TIMESTAMP}-$(basename "${INPUT_FILE}" | sed 's/\.[^.]*$//')"
    JOB_FILE="${REPO_ROOT}/queue/pending/${JOB_NAME}.yaml"

    cat > "${JOB_FILE}" <<EOF
    solver: ${SOLVER}
    input_file: ${INPUT_FILE}
    export_excel: true
    description: "${DESCRIPTION}"
    submitted_by: $(hostname)
    submitted_at: ${TIMESTAMP}
    EOF

    cd "${REPO_ROOT}"
    git add "queue/pending/${JOB_NAME}.yaml"
    git commit -m "queue: submit ${JOB_NAME}"
    git push origin main

    echo "Job submitted: ${JOB_FILE}"
    echo "Queue will process within 30 minutes"
    ```

    **Step 4 — Create `scripts/solver/process-queue.py`:**
    Python script that Claude Code CLI runs on licensed-win-1. Designed to be invoked as:
    `claude -p "Run: python scripts/solver/process-queue.py"`
    OR directly: `python scripts/solver/process-queue.py`

    The script must:
    1. `git pull origin main` to get latest pending jobs
    2. List YAML files in `queue/pending/`
    3. For each job:
       a. Parse the YAML (solver type, input file path)
       b. Move job file to `queue/completed/{job-name}/` (creates dir)
       c. Run OrcFxAPI: load .owd → Calculate() → SaveData() → optional Excel export
       d. Save .owr and .xlsx artifacts into `queue/completed/{job-name}/`
       e. Write a `result.yaml` with status, timing, output paths
       f. If failure: move to `queue/failed/` with error details
    4. `git add queue/` → `git commit` → `git push origin main`

    Important:
    - Use relative paths from repo root, not hardcoded absolutes
    - Handle the case where OrcFxAPI is not importable (fail gracefully, log)
    - Handle the case where queue/pending/ is empty (exit cleanly, no commit)
    - Use `pathlib.Path` throughout
    - Keep it simple — this is a queue processor, not a framework

    **Step 5 — Create `scripts/solver/setup-scheduler.ps1`:**
    PowerShell script for one-time Task Scheduler setup on licensed-win-1:
    ```powershell
    # ABOUTME: One-time setup — creates Windows Task Scheduler job for solver queue polling
    # Run as Administrator on licensed-win-1

    $RepoPath = Split-Path -Parent (Split-Path -Parent (Split-Path -Parent $PSScriptRoot))
    $TaskName = "SolverQueue"
    $PythonScript = Join-Path $RepoPath "scripts\solver\process-queue.py"

    # Create the scheduled task action
    # Pull latest, then process queue
    $Action = New-ScheduledTaskAction -Execute "powershell.exe" -Argument @"
    -NoProfile -ExecutionPolicy Bypass -Command "cd '$RepoPath'; git pull origin main 2>&1; python '$PythonScript' 2>&1 | Out-File -Append '$RepoPath\queue\solver-queue.log'"
    "@

    # Trigger: every 30 minutes, indefinitely
    $Trigger = New-ScheduledTaskTrigger -Once -At (Get-Date) -RepetitionInterval (New-TimeSpan -Minutes 30) -RepetitionDuration ([TimeSpan]::MaxValue)

    # Settings: run whether logged in or not, don't stop if running longer than default
    $Settings = New-ScheduledTaskSettingsSet -AllowStartIfOnBatteries -DontStopIfGoingOnBatteries -StartWhenAvailable -ExecutionTimeLimit (New-TimeSpan -Hours 1)

    # Register (or update if exists)
    if (Get-ScheduledTask -TaskName $TaskName -ErrorAction SilentlyContinue) {
        Set-ScheduledTask -TaskName $TaskName -Action $Action -Trigger $Trigger -Settings $Settings
        Write-Host "Updated existing task: $TaskName"
    } else {
        Register-ScheduledTask -TaskName $TaskName -Action $Action -Trigger $Trigger -Settings $Settings -Description "Polls queue/pending/ for solver jobs every 30 minutes" -RunLevel Highest
        Write-Host "Created task: $TaskName"
    }

    Write-Host ""
    Write-Host "Verify: Get-ScheduledTask -TaskName '$TaskName'"
    Write-Host "Manual run: Start-ScheduledTask -TaskName '$TaskName'"
    Write-Host "Logs: $RepoPath\queue\solver-queue.log"
    ```

    **Step 6 — Make scripts executable:**
    ```bash
    chmod +x scripts/solver/submit-job.sh
    ```
