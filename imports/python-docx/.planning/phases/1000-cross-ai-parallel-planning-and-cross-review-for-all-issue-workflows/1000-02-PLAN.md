**6. `dispatch_plan()` function:**
```bash
dispatch_plan() {
  local provider="$1" tmpdir="$2" prompt_file="$3"
  local output_file="$tmpdir/${provider}-plan.md"
  local err_file="$tmpdir/${provider}-err.log"

  if [[ "$DRY_RUN" == "true" ]]; then
    # Mock output: copy prompt file as plan (simulates a provider that echoes input)
    cp "$prompt_file" "$output_file"
    echo "[DRY-RUN] $provider plan generated" > "$err_file"
    return 0
  fi

  case "$provider" in
    claude)
      claude -p "$(cat "$prompt_file")" --no-input > "$output_file" 2>"$err_file"
      ;;
    codex)
      codex exec --skip-git-repo-check "$(cat "$prompt_file")" > "$output_file" 2>"$err_file"
      ;;
    gemini)
      gemini -p "$(cat "$prompt_file")" > "$output_file" 2>"$err_file"
      ;;
  esac
}
```

**7. Parallel dispatch with PID tracking:**

```bash
declare -A PIDS
echo -e "${CYAN}Dispatching parallel plans to 3 providers...${NC}"

done
```

```bash
done
```

**9. `extract_section()` function** for structured diff (per D-04):
```bash
extract_section() {
  local file="$1" tag="$2"
  # Extract content between <tag> and </tag>, excluding the tags themselves
  sed -n "/<${tag}>/,/<\/${tag}>/p" "$file" | sed '1d;$d'
}

extract_frontmatter() {
  local file="$1"
  # Extract YAML between first --- and second ---
  awk '/^---$/{c++; if(c==2) exit; next} c==1{print}' "$file"
}
```

- `DIVERGENT` — significant differences exist

Build two arrays: `AGREED_SECTIONS` and `DIVERGENT_SECTIONS`.

**11. `merge_plans()` function:**
For AGREED sections: take any provider's version (they're equivalent).
For DIVERGENT sections: build a synthesis prompt and call Claude.

The synthesis prompt for each divergent section:

```
{gemini_section}

3. Combined insights where approaches complement

```

Call synthesis:
