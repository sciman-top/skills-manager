## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Standard-tag pill horizontal padding, icon-bar gaps |
| sm | 8px | Compact element spacing, navbar brand padding, badge gaps |
| md | 16px | Default element spacing, btn horizontal padding, container gutters (15px rounds to 16) |
| lg | 24px | Section padding between stacked elements, form-group margin-bottom (20px rounds to 24) |
| xl | 32px | Card inner padding (30px existing -- use 32px for new components) |
| 2xl | 48px | Page-header bottom margin, footer vertical padding (50px existing -- use 48px for new) |
| 3xl | 64px | Section vertical padding (60px existing -- use 64px for new sections) |

Exceptions:
- Existing pages keep their current pixel values (30px card padding, 60px section padding) to avoid regressions. New pages and new sections use the 8-point scale above.
- Touch targets: minimum 44px height on all interactive elements per existing `responsive.css` (`.btn { min-height: 44px }`).
- Nav links: 48px min-height per existing `responsive.css` (`.navbar-nav > li > a { min-height: 48px }`).

**Source:** Codebase inspection -- `assets/css/marketing.css`, `assets/css/responsive.css`.

## Interaction Patterns

### Calculator form submission:
1. User fills form fields (pre-populated with sensible defaults from manifest)
2. User clicks "Calculate [Metric]" button
3. JavaScript validates inputs (non-zero, within physical ranges)
4. If invalid: `alert()` with specific field name (existing pattern)
5. If valid: calculate result, update `.result-box` value and details table, show `#additionalResults`, render Plotly chart
6. Apply `.result-warning` (red border) or `.result-success` (green border) based on pass/fail threshold
7. Fire GA4 `calculator_use` event with calculator name, key inputs, and result

### Calculator Plotly chart:
- On-bottom stability: line chart showing utilisation ratio vs current velocity sweep (fixed other params). Horizontal line at utilisation = 1.0. Red/green zones above/below.
- Wall thickness: grouped bar chart comparing minimum wall thickness across codes (ASME B31.4, B31.8, API) for given inputs.
- Chart config: `{responsive: true, displayModeBar: false}` to keep clean. Allow Plotly's built-in hover tooltips.

### Pricing page interaction:
- Static display only. No toggles, no sliders, no dynamic pricing.
- Each tier card: hover shadow (same as `.calculator-card:hover`).
- All CTA buttons route to `contact.html`.

### Navigation additions:
- "Calculators" nav link: standard `<li><a href="...">` in `.navbar-nav`. No dropdown submenu.
- "Request Pricing" in `navbar-right`: replace existing "Get a Quote" text and href (points to `pricing.html`).

## SEO Contract Per New Page

Every new page must include:
1. Unique `<title>` with primary keyword + " - A&CE"
2. `<meta name="description">` under 160 characters with primary keyword
3. `<meta name="keywords">` with 5-8 terms
4. Open Graph: `og:title`, `og:description`, `og:type`, `og:url`, `og:site_name`
5. Schema.org `WebApplication` JSON-LD (calculators) or appropriate type (pricing)
6. Entry in `sitemap.xml` with `<lastmod>` date
7. Internal links from calculator index page
8. H1 containing primary keyword
9. Illustrative disclaimer (D-06)
10. CTA to contact for full analysis

**Source:** D-10, D-11, D-12. Existing calculator pages establish this exact pattern.
