name: governance-teaching-lite-output
description: Use when governance tasks need delivery plus concise teaching alignment (结论->原因->下一步) under token budget constraints.
---

1. Default to `lite` output mode for governance collaboration: first deliver executable result, then add minimum teaching context.
2. Use one-pass structure in Chinese: `结论 -> 原因 -> 下一步`; keep each line short and decision-oriented.
3. Before major action, restate mapping in one sentence: `我在做X，为了Y，验收看Z` to align intent and acceptance.
4. When new terms appear, explain once with `一句话定义 + 本仓示例 + 常见误解`; avoid repeating full definitions later.
5. For debugging/triage, use observable chain: `现象 -> 假设 -> 验证命令 -> 预期结果 -> 下一步`.
6. Mark key assumptions as `已确认/未确认`; if continuing with unconfirmed assumptions, include risk and rollback trigger.
7. Respect token budget: avoid long background dumps unless user explicitly asks for deep explanation.
