# Skills Manager 结构性重构设计（入口脚本）

日期：2026-02-01

## 目标
- 提升可维护性、稳定性与性能，不依赖外部工具。
- 交互流程更一致，错误提示更清晰，危险操作可预览且可回滚。
- 保持核心功能与用户心智不变，仅在提示与保护上更严格。

## 约束
- 不修改 `vendor/`、`agent/` 生成目录的内容与结构。
- 不引入新依赖，不改变外部数据格式（`skills.json`）字段含义。
- 仅调整交互流程与内部结构，允许更严格的输入校验与确认。

## 总体方案
将 `skills.ps1` 抽象为四层，并用统一管线驱动各功能：

1) 输入解析层
- 统一 CLI 与菜单路径的参数解析（`menu/add/npx`）。
- 统一选择器解析（序号/范围/全选/取消）。
- 统一路径归一化与目标解析（`~`、相对路径、分隔符）。

2) 配置与状态层
- 统一 `skills.json` 读写、归一化、修复与校验。
- 维护轻量缓存并在配置变更后失效。
- 变更前后差异摘要（便于预览与回滚）。

3) 操作层
- Git/robocopy/mklink/文件操作安全封装与错误一致处理。
- 统一危险动作的“预演-确认-执行”节奏。

4) 展示层
- 统一日志格式与终端提示，输出变更摘要与风险提示。

## 流程统一化
把“发现/安装/卸载/构建/更新/清理/解除关联”视为同一管线：
- 校验 → 预览（影响范围与样例） → 确认 → 执行 → 汇总。

## 交互改进重点
- “安装/卸载/清理备份/解除关联”等危险操作先预览，再确认。
- “导入/add/npx”统一解析链，减少分支差异。
- 明确提示“将要写入的配置键/目标目录/覆盖范围”。

## 性能与鲁棒性
- 扫描类操作限制到文件（避免目录遍历噪声）。
- 排除生成目录与上游缓存的全量扫描。
- 配置字段类型错误时自动修复或明确报错。

## 最小验证
- `发现`（含过滤）
- `安装/卸载`（写入白名单与构建路径）
- `构建生效`（生成 agent 并同步 targets）
- `更新`（拉取上游并重建）

## 回滚策略
- `skills.json` 写入前备份，失败则回滚原始内容。
- 对备份清理类操作输出预览清单。
