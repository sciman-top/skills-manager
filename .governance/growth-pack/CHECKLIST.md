# Growth Pack Checklist

## Minimum Conversion Baseline
- README explains the value proposition in the first screen
- README includes Quick Start, one minimal example, and a visible demo or sample path
- Release note template exists and includes upgrade guidance
- Issue and PR templates exist
- A security reporting path is documented
- A GitHub presence playbook exists for repo page, topics, and trial-path guidance
- The repository homepage, README, and releases tell the same story

## Suggested KPI
- README -> clone conversion
- Clone -> first successful run conversion
- New issue first response time (<24h target)
